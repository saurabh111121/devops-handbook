Complete YAML Reference

Every Kubernetes resource YAML template you need, with apiVersion lookup, field explanations, and kubectl generators. Copy-paste ready for the exam and production.

1. YAML Basics

Top-Level Structure

Every Kubernetes manifest has four top-level fields:

basic-structure.yaml Structure 
apiVersion: v1              # API group/version for this resource
kind: Pod                   # Type of resource
metadata:                   # Data that identifies the resource
  name: my-pod             # Required: unique within namespace
  namespace: default       # Optional: defaults to "default"
  labels:                  # Key-value pairs for selection/grouping
    app: web
    env: production
  annotations:             # Non-identifying metadata (tools, configs)
    description: "Main web pod"
spec:                       # Desired state (resource-specific)
  containers:
  - name: web
    image: nginx:1.25

Indentation Rules

Critical Rules

  • 2 spaces per indentation level (standard for Kubernetes)
  • NO tabs - YAML does not allow tab characters
  • Indentation defines hierarchy/nesting
  • Consistent indentation within same block is mandatory
  • A child element must be indented more than its parent

Multi-line Strings

multi-line-strings.yaml Strings 
# Literal Block Scalar (|) - preserves newlines exactly
data:
  config.ini: |
    [database]
    host=localhost
    port=5432
    name=mydb

# Folded Block Scalar (>) - folds newlines into spaces
data:
  description: >
    This is a long description
    that will be folded into
    a single line with spaces.

# Literal with strip (|-) - removes trailing newline
data:
  script: |-
    #!/bin/bash
    echo "hello"

# Literal with keep (|+) - keeps all trailing newlines
data:
  text: |+
    line one
    line two

Lists vs Maps

lists-and-maps.yaml Data Types 
# Map (dictionary/object) - key: value pairs
metadata:
  name: my-app
  namespace: production

# List (array) - items prefixed with dash
containers:
- name: app
  image: nginx
- name: sidecar
  image: envoy

# List of scalars
args:
- "--config=/etc/app/config.yaml"
- "--verbose"
- "--port=8080"

# Inline list (flow sequence)
ports: [80, 443, 8080]

# Inline map (flow mapping)
labels: {app: web, tier: frontend}

# Boolean values (be careful!)
enabled: true           # boolean true
enabled: "true"         # string "true"
port: 80               # integer
port: "80"             # string

# Null values
value: null
value: ~
value:                 # also null (empty value)

Comments

comments.yaml Comments 
# Full line comment
apiVersion: v1        # Inline comment after value
kind: Pod
metadata:
  name: example
  # Comments can be at any indentation level
  labels:
    app: web          # Labels for pod selection

3. Complete YAML Templates

Pod (Full Featured)

pod-full.yaml Pod 
apiVersion: v1
kind: Pod
metadata:
  name: web-app
  namespace: production
  labels:
    app: web
    version: v2
  annotations:
    prometheus.io/scrape: "true"
    prometheus.io/port: "8080"
spec:
  serviceAccountName: web-sa         # ServiceAccount for RBAC
  restartPolicy: Always              # Always | OnFailure | Never
  terminationGracePeriodSeconds: 30  # Time to wait before SIGKILL

  # Security context at pod level (applies to all containers)
  securityContext:
    runAsUser: 1000                   # UID to run as
    runAsGroup: 3000                  # GID to run as
    fsGroup: 2000                    # GID for volume mounts
    runAsNonRoot: true               # Refuse to run as root

  # Init containers run sequentially before main containers
  initContainers:
  - name: init-db-check
    image: busybox:1.36
    command: ['sh', '-c', 'until nc -z db-svc 5432; do sleep 2; done']
  - name: init-migrations
    image: myapp/migrations:v2
    env:
    - name: DB_URL
      valueFrom:
        secretKeyRef:
          name: db-secret
          key: url

  containers:
  - name: web
    image: myapp/web:v2.1.0
    imagePullPolicy: IfNotPresent    # Always | IfNotPresent | Never

    # Port definitions
    ports:
    - containerPort: 8080
      name: http
      protocol: TCP
    - containerPort: 9090
      name: metrics

    # Environment variables
    env:
    - name: APP_ENV
      value: "production"
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: db-secret
          key: password
    - name: CONFIG_VALUE
      valueFrom:
        configMapKeyRef:
          name: app-config
          key: log-level
    - name: POD_NAME
      valueFrom:
        fieldRef:                     # Downward API
          fieldPath: metadata.name
    - name: CPU_LIMIT
      valueFrom:
        resourceFieldRef:
          resource: limits.cpu

    # All env from ConfigMap/Secret
    envFrom:
    - configMapRef:
        name: app-env-config
    - secretRef:
        name: app-env-secrets

    # Resource requests and limits
    resources:
      requests:
        cpu: 100m                    # 0.1 CPU cores
        memory: 128Mi               # 128 mebibytes
        ephemeral-storage: 1Gi
      limits:
        cpu: 500m                    # 0.5 CPU cores
        memory: 512Mi
        ephemeral-storage: 2Gi

    # Startup probe - gives slow-starting containers time
    startupProbe:
      httpGet:
        path: /healthz
        port: http
      failureThreshold: 30           # 30 * 10s = 5 minutes to start
      periodSeconds: 10

    # Liveness probe - restart if unhealthy
    livenessProbe:
      httpGet:
        path: /healthz
        port: 8080
        httpHeaders:
        - name: X-Health-Check
          value: "true"
      initialDelaySeconds: 15
      periodSeconds: 20
      timeoutSeconds: 5
      failureThreshold: 3
      successThreshold: 1

    # Readiness probe - remove from service if not ready
    readinessProbe:
      httpGet:
        path: /ready
        port: 8080
      initialDelaySeconds: 5
      periodSeconds: 10
      timeoutSeconds: 3
      failureThreshold: 3

    # Container security context
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop: ["ALL"]
        add: ["NET_BIND_SERVICE"]

    # Volume mounts
    volumeMounts:
    - name: config-vol
      mountPath: /etc/app/config
      readOnly: true
    - name: secret-vol
      mountPath: /etc/app/secrets
      readOnly: true
    - name: data-vol
      mountPath: /var/data
    - name: tmp
      mountPath: /tmp

    # Lifecycle hooks
    lifecycle:
      postStart:
        exec:
          command: ["/bin/sh", "-c", "echo started > /tmp/started"]
      preStop:
        exec:
          command: ["/bin/sh", "-c", "sleep 5 && kill -SIGTERM 1"]

  # Sidecar container
  - name: log-shipper
    image: fluent/fluent-bit:2.1
    resources:
      requests:
        cpu: 50m
        memory: 64Mi
      limits:
        cpu: 100m
        memory: 128Mi
    volumeMounts:
    - name: data-vol
      mountPath: /var/data
      readOnly: true

  # Volumes
  volumes:
  - name: config-vol
    configMap:
      name: app-config
      items:                          # Optional: select specific keys
      - key: app.conf
        path: application.conf
  - name: secret-vol
    secret:
      secretName: app-secrets
      defaultMode: 0400              # File permissions
  - name: data-vol
    persistentVolumeClaim:
      claimName: app-data-pvc
  - name: tmp
    emptyDir:
      sizeLimit: 100Mi

  # Node scheduling
  nodeSelector:
    disktype: ssd
    kubernetes.io/arch: amd64

  # Affinity rules
  affinity:
    nodeAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        nodeSelectorTerms:
        - matchExpressions:
          - key: topology.kubernetes.io/zone
            operator: In
            values: ["us-east-1a", "us-east-1b"]
      preferredDuringSchedulingIgnoredDuringExecution:
      - weight: 80
        preference:
          matchExpressions:
          - key: node-type
            operator: In
            values: ["high-memory"]
    podAntiAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
      - labelSelector:
          matchExpressions:
          - key: app
            operator: In
            values: ["web"]
        topologyKey: kubernetes.io/hostname   # One pod per node

  # Tolerations (allow scheduling on tainted nodes)
  tolerations:
  - key: "dedicated"
    operator: "Equal"
    value: "web"
    effect: "NoSchedule"
  - key: "node.kubernetes.io/not-ready"
    operator: "Exists"
    effect: "NoExecute"
    tolerationSeconds: 300

  # DNS configuration
  dnsPolicy: ClusterFirst
  dnsConfig:
    nameservers:
    - 8.8.8.8
    searches:
    - my-namespace.svc.cluster.local

  # Image pull secrets (for private registries)
  imagePullSecrets:
  - name: registry-credentials

  # Topology spread constraints
  topologySpreadConstraints:
  - maxSkew: 1
    topologyKey: topology.kubernetes.io/zone
    whenUnsatisfiable: DoNotSchedule
    labelSelector:
      matchLabels:
        app: web

Deployment

deployment.yaml Deployment 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app
  namespace: production
  labels:
    app: web
    version: v2
  annotations:
    kubernetes.io/change-cause: "Updated to v2.1.0"
spec:
  replicas: 3                        # Desired number of pods
  revisionHistoryLimit: 10           # Number of old ReplicaSets to keep

  # How to find pods this deployment manages
  selector:
    matchLabels:
      app: web                       # Must match template labels

  # Update strategy
  strategy:
    type: RollingUpdate              # RollingUpdate | Recreate
    rollingUpdate:
      maxSurge: 1                    # Max pods above desired (absolute or %)
      maxUnavailable: 0              # Max pods unavailable during update

  # Minimum time a pod must be ready before considered available
  minReadySeconds: 10

  # Seconds to wait for rollout before marking as failed
  progressDeadlineSeconds: 600

  # Pod template
  template:
    metadata:
      labels:
        app: web                     # Must match selector.matchLabels
        version: v2
      annotations:
        prometheus.io/scrape: "true"
    spec:
      serviceAccountName: web-sa
      terminationGracePeriodSeconds: 30
      containers:
      - name: web
        image: myapp/web:v2.1.0
        ports:
        - containerPort: 8080
          name: http
        resources:
          requests:
            cpu: 200m
            memory: 256Mi
          limits:
            cpu: "1"
            memory: 512Mi
        livenessProbe:
          httpGet:
            path: /healthz
            port: http
          initialDelaySeconds: 15
          periodSeconds: 20
        readinessProbe:
          httpGet:
            path: /ready
            port: http
          initialDelaySeconds: 5
          periodSeconds: 10
        env:
        - name: APP_ENV
          value: "production"
        volumeMounts:
        - name: config
          mountPath: /etc/app
      volumes:
      - name: config
        configMap:
          name: web-config
      affinity:
        podAntiAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
          - weight: 100
            podAffinityTerm:
              labelSelector:
                matchExpressions:
                - key: app
                  operator: In
                  values: ["web"]
              topologyKey: kubernetes.io/hostname

Service (ClusterIP / NodePort / LoadBalancer)

service-clusterip.yaml Service 
# ClusterIP - internal only (default)
apiVersion: v1
kind: Service
metadata:
  name: web-svc
  namespace: production
  labels:
    app: web
spec:
  type: ClusterIP                    # Default type
  selector:
    app: web                         # Matches pod labels
  ports:
  - name: http
    port: 80                         # Port the service listens on
    targetPort: 8080                 # Port on the pod (can use name)
    protocol: TCP
  - name: metrics
    port: 9090
    targetPort: metrics              # Using named port from pod
  sessionAffinity: None              # None | ClientIP
---
# Headless Service (for StatefulSets / direct pod DNS)
apiVersion: v1

metadata:
  name: db-headless

  type: ClusterIP

    app: database

Ingress

ingress.yaml Ingress 
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: web-ingress
  namespace: production
  annotations:
    # NGINX Ingress Controller annotations
    nginx.ingress.kubernetes.io/rewrite-target: /
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
    nginx.ingress.kubernetes.io/proxy-body-size: "50m"
    nginx.ingress.kubernetes.io/rate-limit: "100"
    nginx.ingress.kubernetes.io/rate-limit-window: "1m"
    # cert-manager annotation for auto TLS
    cert-manager.io/cluster-issuer: "letsencrypt-prod"
spec:
  ingressClassName: nginx            # Which ingress controller to use
  tls:
  - hosts:
    - app.example.com
    - api.example.com
    secretName: app-tls-secret       # TLS cert stored as Secret

  rules:
  # Host-based routing
  - host: app.example.com
    http:
      paths:
      - path: /
        pathType: Prefix             # Prefix | Exact | ImplementationSpecific
        backend:
          service:
            name: frontend-svc
            port:
              number: 80
      - path: /api
        pathType: Prefix
        backend:
          service:
            name: api-svc
            port:
              number: 8080
      - path: /static
        pathType: Prefix
        backend:
          service:
            name: cdn-svc
            port:
              name: http

  - host: api.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: api-svc
            port:
              number: 8080

  # Default backend (catch-all)
  defaultBackend:
    service:
      name: default-svc
      port:
        number: 80

HorizontalPodAutoscaler (HPA)

hpa.yaml HPA 
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: web-hpa
  namespace: production
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: web-app

  minReplicas: 3
  maxReplicas: 50

  metrics:
  # CPU-based scaling
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70       # Scale when avg CPU > 70%

  # Memory-based scaling
  - type: Resource
    resource:
      name: memory
      target:
        type: Utilization
        averageUtilization: 80       # Scale when avg memory > 80%

  # Custom metric (e.g., requests per second)
  - type: Pods
    pods:
      metric:
        name: http_requests_per_second
      target:
        type: AverageValue
        averageValue: "1000"         # Scale when avg RPS > 1000

  # External metric (e.g., SQS queue depth)
  - type: External
    external:
      metric:
        name: sqs_messages_visible
        selector:
          matchLabels:
            queue: "orders"
      target:
        type: AverageValue
        averageValue: "30"

  # Scaling behavior (control scale up/down speed)
  behavior:
    scaleUp:
      stabilizationWindowSeconds: 60   # Wait before scaling up again
      policies:
      - type: Percent
        value: 100                     # Can double pods
        periodSeconds: 60
      - type: Pods
        value: 5                       # Or add max 5 pods
        periodSeconds: 60
      selectPolicy: Max                # Use whichever allows more scaling
    scaleDown:
      stabilizationWindowSeconds: 300  # Wait 5 min before scaling down
      policies:
      - type: Percent
        value: 10                      # Remove max 10% of pods
        periodSeconds: 60
      selectPolicy: Min                # Use whichever removes fewer pods

Role + RoleBinding

rbac.yaml RBAC 
# Role (namespace-scoped permissions)
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: pod-reader
  namespace: production

5. Resource Units Reference

CPU Units

CPU Measurement

  • 1 = 1 vCPU/core (1 AWS vCPU, 1 GCP core, 1 Azure vCore, 1 hyperthread)
  • 1000m = 1 core (m = millicores)
  • 500m = 0.5 cores = half a CPU
  • 100m = 0.1 cores = 100 millicores
  • 250m = 0.25 cores = quarter CPU
  • Minimum allowed: 1m
cpu-examples.yaml CPU 
resources:
  requests:
    cpu: 100m        # 0.1 CPU (good for lightweight sidecars)
    cpu: 250m        # 0.25 CPU (small web apps)
    cpu: 500m        # 0.5 CPU (medium workloads)
    cpu: "1"         # 1 full CPU core (quotes needed for integers)
    cpu: "2"         # 2 CPU cores
    cpu: 1500m       # 1.5 CPU cores
  limits:
    cpu: "4"         # Max 4 CPU cores

Memory Units

Memory Measurement

Binary (power of 2) - Preferred in Kubernetes:

  • Ki = Kibibyte = 1024 bytes
  • Mi = Mebibyte = 1024 Ki = 1,048,576 bytes
  • Gi = Gibibyte = 1024 Mi = 1,073,741,824 bytes
  • Ti = Tebibyte = 1024 Gi

Decimal (power of 10) - Less common:

  • K = Kilobyte = 1000 bytes
  • M = Megabyte = 1000 K = 1,000,000 bytes
  • G = Gigabyte = 1000 M = 1,000,000,000 bytes
  • T = Terabyte = 1000 G

Warning: 128M (megabytes) is NOT the same as 128Mi (mebibytes). 128Mi = 134,217,728 bytes vs 128M = 128,000,000 bytes. Always use Mi/Gi for clarity.

memory-examples.yaml Memory 
resources:
  requests:
    memory: 64Mi      # 64 mebibytes (sidecars, tiny apps)
    memory: 128Mi     # 128 mebibytes (small apps)
    memory: 256Mi     # 256 mebibytes (medium apps)
    memory: 512Mi     # 512 mebibytes
    memory: 1Gi       # 1 gibibyte (1024 Mi)
    memory: 2Gi       # 2 gibibytes
  limits:
    memory: 4Gi       # Max 4 gibibytes
    memory: 8Gi       # Max 8 gibibytes

# Common patterns:
# - Request = what you expect normal usage to be
# - Limit = max before OOMKill
# - Typical ratio: limit = 2x request for memory
# - CPU: often no limit (or high limit) to avoid throttling

Storage Units

storage-examples.yaml Storage 
# PVC storage uses same units as memory
resources:
  requests:
    storage: 1Gi      # 1 gibibyte
    storage: 10Gi     # 10 gibibytes
    storage: 100Gi    # 100 gibibytes
    storage: 1Ti      # 1 tebibyte

# ephemeral-storage (container local storage)
resources:
  requests:
    ephemeral-storage: 1Gi
  limits:
    ephemeral-storage: 5Gi    # Evicted if exceeded

Common Gotchas

  • CPU limit throttling: When a container hits CPU limit, it is throttled (not killed). When it hits memory limit, it is OOMKilled.
  • No CPU limit debate: Many teams set no CPU limit to avoid throttling. Use requests for scheduling, let pods burst.
  • Memory limit = request: For Java/Go apps with fixed heap, set memory limit = request to avoid OOM surprises.
  • QoS classes: Guaranteed (requests=limits), Burstable (requests < limits), BestEffort (no requests/limits). Guaranteed pods are evicted last.
  • Integer CPU: Must be quoted: cpu: "1" not cpu: 1 (YAML interprets bare integers differently in some contexts).
